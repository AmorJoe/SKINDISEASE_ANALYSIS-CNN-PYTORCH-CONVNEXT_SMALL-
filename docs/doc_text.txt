SYNOPSIS 
In the existing system, skin diseases are diagnosed mainly through manual visual
examination by dermatologists. This process depends heavily on the expertise and 
experience of medical professionals and requires patients to visit hospitals or 
clinics. Such procedures can be time-consuming, expensive, and less accessible, 
especially in rural or remote areas. Delays in diagnosis may lead to complications 
and affect timely treatment. 
The proposed system allows users to upload images of affected skin areas through 
a simple and user-friendly interface. The uploaded image undergoes 
preprocessing, including resizing and normalization, before being passed to the 
trained CNN model. The model extracts relevant features, compares them with 
trained dataset patterns, and predicts the most probable disease class. The system 
also generates a confidence score to indicate prediction reliability and provides 
basic care suggestions. 
This project aims to assist in early detection, reduce human error, save diagnostic 
time, and improve healthcare accessibility. While it does not replace professional 
medical consultation, it serves as an effective preliminary screening tool to 
support both patients and healthcare providers.
Skin Disease Detection Using Image Analysis is an intelligent web-based system
developed to detect and classify various skin diseases using image processing and
deep learning techniques. The system focuses on analyzing visual features such as
color, texture, shape, and lesion patterns from skin images to provide a
preliminary diagnosis. A Convolutional Neural Network (CNN) is used as the
core model to automatically extract important features and classify the image into
predefined disease categories.

Skin diseases are among the most common medical conditions affecting people of all age
groups worldwide. Early detection and proper diagnosis are essential to prevent
complications, reduce disease spread, and ensure timely treatment. However, traditional
diagnostic procedures often depend on manual visual examination by dermatologists,
which can be time-consuming, expensive, and less accessible, especially in rural or remote
areas. 
The project "Skin Disease Detection Using Image Analysis" aims to develop an 
intelligent web-based system that detects and classifies various skin diseases using image 
processing and deep learning techniques. The system focuses on analyzing visual features 
such as color, texture, shape, and lesion patterns from skin images to provide a 
preliminary diagnosis. 
A Convolutional Neural Network (CNN) is used as the core model to automatically 
extract important features and classify the image into predefined disease categories. The 
proposed system allows users to upload images of affected skin areas through a simple 
and user-friendly interface. The uploaded image undergoes preprocessing, including 
resizing and normalization, before being passed to the trained CNN model. 
The model extracts relevant features, compares them with trained dataset patterns, and 
predicts the most probable disease class. The system also generates a confidence score to 
indicate prediction reliability and provides basic care suggestions. This project aims to 
assist in early detection, reduce human error, save diagnostic time, and improve 
healthcare accessibility. 
In the existing system, skin diseases are diagnosed mainly through manual visual examination
by dermatologists. This process depends heavily on the expertise and experience of medical
professionals and requires patients to visit hospitals or clinics for consultation. Such
procedures can be time-consuming, expensive, and less accessible, especially in rural or
remote areas. Delays in diagnosis may lead to complications and affect timely treatment.

Fundamental design concepts form the foundation of software development and help in
building reliable, scalable, and maintainable systems. These concepts guide the overall
structure of the application and ensure that the system functions efficiently while

The Skin Disease Detection System is designed using modular architecture. Each module 
performs a specific function, which makes the system easier to develop, test, and 
maintain. The design ensures: 
• 
• 
• 
• 
• 
Proper separation of responsibilities. 
Reduced system complexity. 
Easy scalability and maintenance. 
Efficient data handling.

The system follows a structured approach where the user interface, business logic, and 
database are organized separately. This improves clarity in development and ensures 
smooth interaction between different components of the system. 
System architecture defines the overall structure of the system and describes how different
components interact with each other. The Skin Disease Detection System follows a three-
tier architecture: 
• 
Presentation Layer (Front-End): Provides a user-friendly interface where users can 
upload skin images and view prediction results. It is developed using HTML, 
CSS, and JavaScript. 
Application Layer (Back-End): Handles image uploads, preprocessing, model
loading, and prediction generation. Developed using Python with Django
framework. 
Data Layer (Database): Stores user details, disease information, and prediction 
history. Managed using Supabase.

Normalization is a technique of organizing the data in the database. Normalization is a
systematic approach of decomposing tables to eliminate data redundancy (repetition)
and undesirable characteristics like Insertion, Update and Deletion Anomalies. It is a
multi- step process that puts data into tabular form, removing duplicated data from the
relation tables. 
In our design the tables have been normalized up to Third Normal Form (3NF). 
Normalizations applied during database design are: 
• 
• 
• 
First Normal Form (1NF) 
Second Normal Form (2NF) 
Third Normal Form (3NF) 
If a relation contains composite or multi-valued attributes, it violates first normal form or a
relation is in first normal form if it does not contain any composite or multi-valued attribute.
A relation is in first normal form if every attribute in that relation is a single- valued
attribute. Our application satisfies 1NF. 
The first normal form states that: 
• 
• 
• 
Every column in the table must be unique. 
Separate tables must be created for each set of related data. 
Each table must be identified with a unique column or concatenated columns 
called the primary key. 
No rows may be duplicated. 
No row/column intersections contain a null value. 
If a relation contains partial dependency, it violates second normal form or a relation is in
second normal form if it does not contain any partial dependency. A relation is in second
normal form if it satisfies first normal form and all non-key attributes are fully dependent
on the primary key. Our application satisfies 2NF.
